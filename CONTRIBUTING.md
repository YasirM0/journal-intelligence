# Contributing

Thank you for your interest in contributing to Journal Intelligence.

Journal Intelligence is an open-source project dedicated to helping researchers make better publishing decisions through transparent and explainable journal recommendations.

We welcome contributions of all sizes, including bug reports, feature suggestions, documentation improvements, and code contributions.

---

# Before Contributing

Please read the following documents before submitting major contributions:

- README.md
- VISION.md
- PROJECT_PRINCIPLES.md
- ARCHITECTURE.md

These documents define the project's goals and design philosophy.

---

# Development Philosophy

Contributions should follow the project's guiding principles:

- Simplicity before complexity
- Transparency over black boxes
- Evidence before assumptions
- Privacy by design
- Modular architecture

If a proposed change conflicts with these principles, it should be discussed before implementation.

---

# Reporting Issues

When reporting a bug, please include:

- Description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

---

# Feature Requests

Feature requests should clearly explain:

- The problem being solved
- Why the feature is valuable
- How it aligns with the project's goals

Preference is given to features that improve transparency, usability, or recommendation quality.

---

# Pull Requests

Before submitting a pull request:

- Ensure the change addresses a specific issue.
- Keep changes focused and well documented.
- Update documentation when necessary.
- Follow the existing project structure.

---

# Code of Conduct

Please be respectful and constructive.

Journal Intelligence aims to build a welcoming community where researchers and developers can collaborate effectively.

---

Thank you for helping improve Journal Intelligence.
